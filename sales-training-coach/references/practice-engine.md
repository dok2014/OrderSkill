# 销售场景推演引擎

## 三层推演体系

### 第一层：角色扮演 — 模拟客户方不同角色

**流程**:
1. 用户提供客户背景（行业/规模/决策链/当前阶段）
2. AI扮演指定角色提问/质疑：
   - CTO/技术负责人：技术实现、方案成熟度、兼容性
   - 采购/商务负责人：价格、付款条件、服务条款
   - 业务负责人：ROI、效果验证、落地可行性
   - 决策层/老板：战略价值、风险控制、长期合作
3. 用户回复后，AI参照 OrderFollower 对应方法论文件评估回答质量并给出优化建议
   - 涉及报价/商务 → 参照 `04-exit-decision.md`
   - 涉及技术沟通 → 参照 `03-sales-presales-collaboration.md`
   - 涉及异议处理 → 参照 `05-professional-commitment.md`
   - 涉及综合场景 → 参照 `02-sales-assistant.md` SOP流程
4. 可切换角色继续，直到用户满意

### 第二层：压力测试 — 模拟最坏情况

**流程**:
1. 用户指定压力场景：
   - 客户当场质疑价格过高
   - 被问到没准备的技术细节
   - 竞品突然降价/推出新功能
   - 客户方的Champion突然离职
   - 客户要求限时内完成不可能的任务
2. AI持续施压制造紧张感，让用户"脱敏"
3. 训练应对突发状况的反应速度和质量

### 第三层：多路径推演 — 不同策略的结果分支

**流程**:
1. 用户给出当前客户状态和可选策略
2. AI模拟2-3条不同路径的可能结果：
   - 路径A（激进报价）：可能结果 + 风险点
   - 路径B（稳扎稳打）：可能结果 + 风险点
   - 路径C（另辟蹊径）：可能结果 + 风险点
3. 用户选择一条路径后，AI模拟后续3-5轮交互
4. 复盘：哪条路径最优？为什么？
   - 复盘时参照 OrderFollower 全流程（01-05）方法论，逐项对比各路径的框架符合度

## 执行节奏

| 类型 | 时长 | 频率 | 适用场景 |
|------|------|------|----------|
| 口播推演 | 15分钟 | 每日08:30 | 修正当日话术，记录待办 |
| 正式推演 | 30分钟 | 每周五16:00 | 专项复盘，调整下周策略 |
| 专项推演 | 20-30分钟 | 关键节点前24h | 完善方案或话术 |
| 复盘推演 | 30分钟 | 输单后 | 提炼可复用模式 |

## 输单复盘推演

当输单发生时执行，**复盘结论纳入 OrderCoach 学习追踪体系，评估标准以 OrderFollower 方法论为基准**：

1. **事实还原**：从最初接触→方案交付→商务谈判→最终结果的完整时间线
   - 涉及阶段判断 → 参照 `01-opportunity-evaluation.md`
2. **关键决策点识别**：找出3-5个关键转折点
   - 涉及售前节点 → 参照 `03-sales-presales-collaboration.md` 会前检查清单
3. **竞争对比**：我们在哪些环节输了？哪些赢了？
   - 参照 `02-sales-assistant.md` 竞争态势洞察
4. **可复用模式提炼**：这次经验中，哪些是下次可以复用的？
5. **改进清单**：下次遇到类似情况，什么必须改？
   - 涉及退出判断 → 参照 `04-exit-decision.md`
   - 涉及客户关系 → 参照 `05-professional-commitment.md`

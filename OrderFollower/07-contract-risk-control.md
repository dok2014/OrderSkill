# 合同风控与审批推动

## 🎯 技能定位：防止死在最后一公里

很多大单在"握手承诺"后死在法务审核或流程卡壳上。双方口头达成一致不等于签约——客户内部还需要走法务、合规、预算审批等多道流程，销售方内部也可能有价格或条款审批。

本阶段在专业承诺之后、签约完成之前，主动推动双方内部流程，将口头承诺转化为有约束力的书面文件。

### 触发条件
当出现以下任一情况时，必须调用本技能：
- 双方已口头达成合作意向，但合同迟迟未签
- 客户说"在走流程"，但超过约定时间无进展
- 法务审核中提出大量修改意见，谈判陷入拉锯
- 客户方突然增加新的审批环节或审批人

---

## 一、核心逻辑：为什么必须单独列为一个阶段

```
专业承诺（握手承诺）
       ↓
  ⚠️ 危险区：被动等待客户内部流程
       ↓
  ❌ 流程卡壳、法务推翻、预算冻结、领导换人
       ↓
  丢单（死在最后一公里）
```

```
专业承诺（握手承诺）
       ↓
  合同风控与审批推动 ← 本阶段
       ↓
  ✅ 主动推动流程、预判风险、锁定时间节点
       ↓
  签约完成
```

**核心原则**：
> 客户说"我去走流程"的时候，你的工作才刚刚开始。流程不会自己走完，需要你主动推动。

## 二、标准动作三步走

### 第一步：帮助客户推动内部审批

**审批路径摸底话术**：

> "X总，接下来您内部需要走哪几步流程？每一步大概需要多长时间？需要我提供什么材料来配合？"

**客户内部审批路径图模板**：

```
┌────────────────────────────────────────────┐
│          客户内部审批路径图                   │
├────────┬──────────┬──────────┬─────────────┤
│  步骤  │  审批人   │  预计耗时 │  需要材料    │
├────────┼──────────┼──────────┼─────────────┤
│ 1.技术 │ 技术总监  │  3个工作日│ 技术方案说明书 │
│   评审 │          │          │ 成功案例     │
├────────┼──────────┼──────────┼─────────────┤
│ 2.业务 │ 业务VP   │  5个工作日│ 成本收益分析表 │
│   评审 │          │          │ ROI测算      │
├────────┼──────────┼──────────┼─────────────┤
│ 3.采购 │ 采购部   │  7个工作日│ 报价单       │
│   评审 │          │          │ 竞品对比表   │
├────────┼──────────┼──────────┼─────────────┤
│ 4.法务 │ 法务部   │ 10个工作日│ 合同草案     │
│   审核 │          │          │ 条款说明     │
├────────┼──────────┼──────────┼─────────────┤
│ 5.最终 │ CEO/CFO  │  5个工作日│ 汇总审批材料  │
│   批准 │          │          │ 汇报PPT     │
├────────┴──────────┴──────────┴─────────────┤
│ 预计总耗时：30个工作日                       │
│ 关键风险点：法务审核（最易卡壳）              │
└────────────────────────────────────────────┘
```

**主动提供的辅助材料清单**：

| 材料 | 用途 | 给谁 |
|------|------|------|
| 技术方案说明书 | 技术评审 | 技术总监/CTO |
| 成功案例/客户证言 | 降低决策风险感知 | 业务VP/决策层 |
| 成本收益分析表/ROI测算 | 证明投资回报 | 业务VP/CFO |
| 竞品对比分析表 | 证明选型合理性 | 采购部/决策层 |
| 向上汇报PPT（协助起草） | 帮助Champion推动内部审批 | Champion → 决策层 |
| 实施计划与风险预案 | 证明落地可行性 | 技术/业务负责人 |

### 第二步：明确签约时间节点，签署预签约备忘录

**预签约备忘录/意向书核心要素**：

```
┌────────────────────────────────────────────┐
│          预签约备忘录 / 合作意向书            │
├────────────────────────────────────────────┤
│ 甲方（客户）：                              │
│ 乙方（我方）：                              │
├────────────────────────────────────────────┤
│ 一、合作意向                                │
│   双方就XXX项目达成合作意向，核心条款如下：    │
│   - 项目范围：                              │
│   - 核心交付物：                            │
│   - 预估金额：                              │
│   - 付款方式（初步）：                       │
├────────────────────────────────────────────┤
│ 二、排他性约定（可选）                       │
│   自本备忘录签署之日起XX个工作日内，           │
│   甲方承诺不就本项目与其他供应商进行实质性谈判   │
├────────────────────────────────────────────┤
│ 三、签约时间节点                            │
│   - 合同草案提交：____年__月__日             │
│   - 法务审核完成：____年__月__日             │
│   - 正式签约日期：____年__月__日             │
├────────────────────────────────────────────┤
│ 四、违约条款（可选）                         │
│   若因甲方原因未在约定时间内完成签约，          │
│   乙方有权调整报价或资源配置                  │
├────────────────────────────────────────────┤
│ 甲方签字：________  乙方签字：________       │
│ 日期：____________  日期：____________       │
└────────────────────────────────────────────┘
```

**时间节点设定的原则**：
- 每个节点必须有明确的截止日期，不能是"尽快""争取"
- 节点之间留合理缓冲（法务审核至少留10个工作日）
- 设定"预警线"：超过约定时间50%仍未完成，启动升级机制

### 第三步：法务条款单独处理，不与价格谈判混在一起

**法务条款拉锯的常见焦点与应对**：

| 条款类型 | 常见争议 | 我方底线 | 谈判策略 |
|----------|----------|----------|----------|
| 付款方式 | 客户要求验收后付款 | 至少30%预付款 | 用实施计划证明分阶段付款的合理性 |
| 验收标准 | 客户要求"完全满足需求" | 明确可量化的验收指标 | 引用方案共识确认纪要中的技术指标 |
| 违约赔偿 | 客户要求高额违约金 | 赔偿上限不超过合同金额 | 区分"直接损失"和"间接损失" |
| 知识产权 | 客户要求全部源码/知识产权 | 保留核心知识产权，授权使用 | 区分"项目定制部分"和"产品平台部分" |
| 售后服务 | 客户要求无限期免费维护 | 明确服务等级和期限 | 提供分档服务方案供选择 |

**法务谈判核心原则**：
> 法务条款谈判和商务价格谈判必须分开进行。混在一起谈，客户会用"条款让步"来压价，你会陷入被动。

**法务谈判话术**：
> "商务条款我们之前已经达成一致了。法务层面的条款我们单独过一遍，确保双方的法律风险都得到合理控制。这两块分开处理，效率更高。"

## 三、常见风险与应对

| 风险场景 | 预警信号 | 应对策略 |
|----------|----------|----------|
| 流程无限拖延 | 超过约定时间2周仍无进展 | 升级接触层级，直接联系决策者了解真实原因 |
| 法务突然推翻核心条款 | 法务提出大量修改，态度强硬 | 区分"原则性问题"和"措辞问题"，原则性问题请双方高层直接沟通 |
| 预算被冻结/削减 | 客户方突然提出重新谈价格 | 回到方案共识确认纪要，强调价值已对齐，探讨分期或缩减范围的替代方案 |
| 审批人变更 | 客户方换了新的审批人 | 第一时间约见新审批人，重新做一次简要的方案价值沟通 |
| 竞品截胡 | 客户突然冷淡，信息回复变慢 | 通过教练确认是否竞品介入，评估是否需要调整策略或退出 |
| 客户要求"先干活后签约" | 客户以"项目紧急"为由要求提前启动 | 坚决拒绝。可签署"启动工作函"限定前期工作范围，但核心交付必须在签约后 |

## 四、阶段检查清单

在进入签约完成前，逐项确认：

- [ ] 客户内部审批路径已摸清（每一步的审批人、耗时、所需材料）
- [ ] 已主动提供辅助审批材料（技术方案、成功案例、ROI测算、汇报PPT等）
- [ ] 预签约备忘录/意向书已签署，明确了签约时间节点
- [ ] 法务条款已单独完成1-2轮沟通，核心分歧已识别
- [ ] 价格条款与法务条款未混在一起谈判
- [ ] 教练已确认客户内部无新增反对者或流程障碍
- [ ] 已设定预警机制：超过约定时间50%自动升级

> 🕵️ **教练验证**：请教练确认审批路径是否完整（有无遗漏的隐性审批人），以及各审批人的真实态度和可能的阻力点。

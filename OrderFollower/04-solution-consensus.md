# 方案共识确认：技术/商务双通过

## 🎯 技能定位：堵死后期推翻方案的后路

大客户销售中，80%的谈判僵局源于"双方对方案范围的理解不一致"。方案展示后客户不置可否就直接进入评估决策，后期谈判才发现要推翻重来——这是最常见的跟单陷阱。

本阶段在售前协同与退出决策之间，强制插入"方案共识确认"环节，用书面留痕锁定双方对方案范围与价值预期的一致理解。

### 触发条件
当出现以下任一情况时，必须调用本技能：
- 方案演示/技术交流已完成，客户未给出明确反馈
- 客户说"方案我们内部再讨论一下"后超过3个工作日无回应
- 即将进入商务谈判阶段，但方案范围尚未书面确认
- 客户方有新的技术把关者加入，可能推翻之前结论

---

## 一、核心逻辑：为什么必须单独列为一个阶段

```
售前协同（展示方案）
       ↓
  ⚠️ 危险区：客户不置可否，销售以为"没问题"
       ↓
  评估决策（谈判才发现理解不一致）
       ↓
  ❌ 推翻重来，售前努力白费
```

```
售前协同（展示方案）
       ↓
  方案共识确认（书面留痕）← 本阶段
       ↓
  评估决策（分歧只在商务条款上）
       ↓
  ✅ 方案已锁定，谈判有据可依
```

**核心原则**：
> 方案确认不是"让客户满意"，而是"让双方对满意标准达成一致并留下证据"。

---

## 二、标准动作三步走

### 第一步：明确提问，确认方案是否解决核心问题

方案展示结束后，不要问"您觉得怎么样？"（太模糊），而要逐项确认：

| 确认维度 | 标准提问话术 | 目的 |
|----------|-------------|------|
| 痛点覆盖 | "基于我们刚才的方案，是否解决了您最关心的X、Y、Z问题？" | 确认方案与痛点对齐 |
| 技术可行性 | "从技术实现角度，有没有哪个部分您觉得落地有难度？" | 暴露潜在技术分歧 |
| 范围边界 | "我们确认一下，本期范围包含A、B、C，不包含D、E，对吗？" | 锁定范围，防止后期范围蔓延 |
| 价值预期 | "按这个方案落地，您预期的业务指标改善大概是什么量级？" | 对齐价值预期，防止后期"效果不达预期" |
| 竞品对比 | "和我们方案相比，您还在评估的其他方案主要差异在哪里？" | 了解竞争态势，预判可能的攻击点 |

**关键技巧**：
- 每个问题都要得到具体回答，不接受"挺好的""差不多"等模糊反馈
- 如果客户说"有个地方需要调整"，当场记录并确认调整后的预期
- 如果客户说"需要内部讨论"，约定具体回复时间（不超过3个工作日）

### 第二步：输出会议纪要，请关键人书面确认

**纪要必备要素**：

```
┌─────────────────────────────────────────┐
│          方案共识确认纪要                 │
├─────────────────────────────────────────┤
│ 会议时间：                               │
│ 参会人员（客户方）：                      │
│ 参会人员（我方）：                        │
├─────────────────────────────────────────┤
│ 一、已确认的方案要点                      │
│   1. 核心需求理解（逐条列出）              │
│   2. 方案覆盖范围（明确边界）              │
│   3. 技术路线与关键指标                   │
│   4. 预期业务价值                        │
├─────────────────────────────────────────┤
│ 二、待确认/待调整事项                     │
│   1. XXX 需进一步确认（责任人+截止时间）    │
│   2. XXX 需补充材料（责任人+截止时间）      │
├─────────────────────────────────────────┤
│ 三、下一步计划                            │
│   1. 客户内部评审（截止时间）              │
│   2. 商务条款沟通（预计时间）              │
│   3. 签约目标时间                         │
├─────────────────────────────────────────┤
│ 确认方式：□ 邮件回复  □ 签字确认          │
│ 确认人（客户方技术/业务负责人）：          │
│ 确认日期：                               │
└─────────────────────────────────────────┘
```

**确认方式优先级**：
1. **最佳**：客户方技术负责人邮件回复"确认，无异议"
2. **次佳**：客户方在会议纪要纸质版上签字
3. **可接受**：客户方在工作群中明确回复"方案没问题"
4. **不可接受**：口头确认、无记录、"应该没问题"

### 第三步：锁定方案范围，堵死后期推翻的后路

**锁定话术模板**：

> "X总，这份纪要确认了我们双方对方案范围和技术路线的一致理解。后续商务谈判中，如果涉及方案范围的调整，我们会基于这份纪要来做变更评估。这样能保证咱们双方都在同一个基准上沟通，避免后期扯皮。"

**锁定后的防御价值**：
- 客户后期若在价格上施压："方案已确认，价值已对齐，分歧只在商务条款上"
- 客户后期若要求增加范围："这部分超出了已确认的方案范围，需要单独评估工作量"
- 新加入的反对者质疑方案："这是X总已确认的方案纪要，我们可以基于此讨论"

## 三、常见阻力与应对

| 客户反应 | 潜在风险 | 应对策略 |
|----------|----------|----------|
| "不用这么正式吧，我们口头确认就行" | 后期可能推翻，无据可依 | "这是我们的标准流程，为了保护双方利益，避免后期理解偏差" |
| "纪要我先看看，回头回复"（超过3天无回复） | 客户内部有分歧，或在拖延 | 主动跟进："X总，纪要中是否有需要调整的地方？如果没问题，麻烦确认一下我们好推进下一步" |
| "方案还需要XX领导看一下" | 决策链未完全覆盖 | "需要我配合准备一份给XX领导的简要说明吗？"主动帮助推动内部沟通 |
| "方案没问题，但预算可能要调整" | 商务分歧开始浮现 | "方案确认和商务谈判我们分开处理。先把方案锁定，商务层面我们后续专项沟通" |

## 四、阶段检查清单

在进入下一阶段（退出决策/专业承诺）前，逐项确认：

- [ ] 已逐项确认方案是否解决客户核心问题
- [ ] 方案范围边界已明确（包含什么、不包含什么）
- [ ] 技术路线和关键指标已对齐
- [ ] 预期业务价值已量化
- [ ] 会议纪要已发送给客户方关键人
- [ ] 已获得客户方书面确认（邮件/签字/工作群明确回复）
- [ ] 待确认事项已明确责任人和截止时间
- [ ] 教练已确认方案在客户内部无重大异议

> 🕵️ **教练验证**：请教练确认方案在客户内部是否还有未覆盖的反对者，以及技术把关者的真实态度。
